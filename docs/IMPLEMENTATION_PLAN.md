# 代码实现方案

## 1. 当前实现范围
本仓库先提供MVP策略评分内核，聚焦截图中的“炒股高手视角”和“基金经理视角”核心逻辑：
- 股票多因子规则评分。
- AND/OR组合模式。
- 必选规则与权重评分。
- ETF筛选策略模板。
- 可解释的命中/未命中原因。

## 2. 分层设计
- `models.py`：定义股票指标、策略规则、策略配置和结果对象。
- `selection_engine.py`：执行规则匹配、评分归一化和候选排序。
- `strategies.py`：沉淀默认股票策略和ETF策略模板。
- `tests/`：验证核心策略引擎行为。

## 3. 后续工程化演进
1. 将当前Python策略内核封装为后端服务API。
2. 接入真实行情和财务数据源。
3. 增加回测执行器和任务队列。
4. 建设移动端Flutter项目，调用策略API并展示结果解释。
5. 增加用户、策略保存、自选股、通知和订阅模块。

## 4. 关键边界
- 当前MVP不连接真实交易系统。
- 当前MVP不承诺收益，只输出研究辅助结果。
- 默认策略仅作为模板，生产环境应允许用户调整阈值与权重。
