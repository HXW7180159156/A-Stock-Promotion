# 技术架构及方案

## 1. 总体架构
```text
移动端/桌面端
  ├─ 策略配置、选股结果、个股详情、ETF详情
  ↓ REST/WebSocket/GraphQL
API网关
  ├─ 鉴权、限流、审计、风险提示
  ↓
业务服务层
  ├─ 选股策略服务
  ├─ ETF筛选服务
  ├─ 回测服务
  ├─ 组合配置服务
  ├─ 风控服务
  └─ 通知服务
  ↓
数据与计算层
  ├─ 实时行情接入
  ├─ 基本面/财务数据
  ├─ 情绪与资金流数据
  ├─ 因子计算任务
  └─ 回测任务队列
  ↓
存储层
  ├─ PostgreSQL：用户、策略、组合、任务元数据
  ├─ InfluxDB/TimescaleDB：时序行情与因子
  ├─ Redis：缓存、排行榜、WebSocket状态
  └─ SQLite：移动端本地缓存
```

## 2. 端侧方案
- **优先建议**：Flutter，用一套代码覆盖iOS、Android、Web和桌面端，适合快速交付MVP。
- **备选方案**：Kotlin Multiplatform适合重业务逻辑共享；React Native适合前端生态复用；MAUI适合.NET团队。
- **端侧缓存**：SQLite保存自选股、最近策略和离线结果；行情类数据设置短TTL。

## 3. 后端方案
- **API层**：RESTful用于常规CRUD；WebSocket用于行情和策略任务状态推送；GraphQL可用于复杂看板聚合查询。
- **服务语言**：MVP可使用Python快速实现策略与回测原型；高并发行情推送可逐步迁移到Go服务。
- **任务调度**：定时任务计算日线因子；异步队列执行回测和参数优化。
- **数据接入**：AkShare或合规数据源提供行情、财务、ETF和资金流数据。

## 4. 核心模块
### 4.1 股票选股策略服务
- 技术指标：均线、MACD、KDJ、RSI、布林带、量价分析。
- 基本面：PE、PB、ROE、营收增速、负债率。
- 情绪面：北向资金、龙虎榜、板块轮动、涨停分析。
- 组合逻辑：AND/OR、权重评分、排序、解释。

### 4.2 ETF服务
- ETF筛选：跟踪误差、流动性、规模、费率、折溢价率。
- 组合配置：行业轮动、大类资产配置、再平衡。
- 风控：最大回撤、夏普比率、波动率、VaR。

### 4.3 回测系统
- 输入：策略规则、标的池、调仓频率、交易成本、时间范围。
- 输出：收益曲线、回撤、波动率、胜率、换手率、样本外表现。
- 防偏差：处理未来函数、幸存者偏差、停牌和涨跌停。

## 5. 数据模型概要
- `StockMetrics`：股票代码、名称、技术面、基本面、情绪面和风险指标。
- `StrategyRule`：指标、比较符、阈值、权重、是否必选。
- `StrategyProfile`：策略名称、组合模式、规则集合。
- `SelectionResult`：总分、命中规则、未命中规则、是否入选。

## 6. 安全与合规
- 所有接口需要鉴权和访问频率限制。
- 数据源授权信息不得进入代码仓库。
- 投资相关页面必须展示风险提示和免责声明。
- 记录策略执行审计日志，便于复盘与合规检查。
